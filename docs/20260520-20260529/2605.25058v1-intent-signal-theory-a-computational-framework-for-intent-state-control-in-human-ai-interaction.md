## 摘要
当前的人工智能交互模型将提示（prompt）视为主要的交换对象，忽略了一个关键层面：用户的潜在源意图（latent source intent），即提示之前并激发提示的目标状态。本文提出意图信号理论（Intent Signal Theory, IST），这是一个形式化描述这一缺失意图层的计算框架。IST 区分了四个经常被混淆的对象：潜在源意图（I*）、可观察的意图代理（I-hat）、编码载体（P）和模型输出（O）。它形式化定义了维度权重、编码掩码、结构和保真度恢复分数，以及公私意图分解。不可逆意图损失定理表明，在载体中缺失的私有意图无法通过通用替换恢复。来自四项伴随研究的证据（涵盖六个大语言模型、三种语言和三个任务领域）显示了与 IST 预测一致的结构-保真度分裂、人工验证的度量分离以及权重容忍平台。IST 将提示工程重新定义为意图协议设计，并识别出当前 AI 系统缺乏的一个计算层面。

## Abstract
Current AI interaction models treat the prompt as the primary object of exchange, omitting a critical layer: the user's latent source intent, the goal state preceding and motivating the prompt. Here we introduce Intent Signal Theory (IST), a computational framework that formalises this missing intent layer. IST distinguishes four objects routinely conflated: latent source intent (I*), observable intent proxy (I-hat), encoded carrier (P), and model output (O). It formalises dimensional weights, encoding masks, structural and fidelity recovery scores, and public-private intent decomposition. The Theorem of Irreversible Intent Loss establishes that private intent absent from the carrier cannot be recovered beyond generic substitution. Evidence from four companion studies spanning six LLMs, three languages and three task domains shows structural-fidelity splits, human-validated metric dissociation, and weight-tolerance plateaus consistent with IST's predictions. IST reframes prompt engineering as intent-protocol design and identifies a computational layer that current AI systems lack.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：当前人机交互（特别是大语言模型）的研究范式将交互视为从提示（prompt）文本到模型输出的映射，忽略了用户潜在的源意图（source intent）——即用户撰写提示之前所持有的目标状态。这种忽视导致系统可能输出结构完整但意图缺失的响应，而现有的评估方法（如整体质量评分）无法检测这种失败。例如，用户意图指定企业客户，但模型输出了面向消费者的市场分析，此时输出结构优秀但意图有缺陷。这一差距是系统性的，源于用户难以完全表达目标状态、模型依赖隐含推理以及评估工具只针对提示表面而非潜在意图。
- **研究含义**：论文提出意图信号理论（Intent Signal Theory, IST），旨在为这一缺失的意图层提供一个统一的计算框架，使得意图的表示、编码、恢复、测量和审计变得可计算、可审计，从而将提示工程重新定义为意图协议设计，为人机交互的可治理性奠定理论基础。

## 2. 方法论：核心思想、关键技术细节、公式或算法流程

### 核心思想
- IST 将单轮人机交互建模为意图状态传输与控制过程：**I*** → **Ĩ** → **P** → **O**，其中：
  - **I***：潜在源意图（固定参考对象，不可直接观察）
  - **Ĩ**：可观察意图代理（结构化的、可操作维度的近似表示）
  - **P**：编码载体（实际输入的提示）
  - **O**：模型输出
- 每个链路都是潜在信息丢失、权重不匹配或模型先验部分补偿的场所。

### 关键技术细节
1. **维度分解与任务条件权重**：对于给定任务类型T，可观察代理表示为加权维度集合：Ĩ = {(eᵢ, wᵢ)}，权重wᵢ≥0且Σwᵢ=1。任务不同，维度重要性不同，权重反映任务条件性。例：5W3H/PPS框架（What, Why, Who, When, Where, How-to-do, How-much, How-feel）是其中的一种高覆盖实例化。
2. **编码掩码与编码损失**：掩码mᵢ∈{0,1}表示维度eᵢ是否明确编码到载体中。编码损失：**L_enc = 1 - Σwᵢmᵢ**，衡量任务加权意图在提示中缺失的比例。
3. **结构恢复与保真度恢复**：
   - 结构恢复分数 rᵢ：输出填补维度插槽的程度，不论内容是否为用户期望。
   - 保真度恢复分数 fᵢ：输出复现用户特定意图内容的程度。
   - 加权聚合：**s-ICMw = Σwᵢrᵢ**，**f-ICMw = Σwᵢfᵢ**。
   - 结构-保真度分裂：s-ICMw >> f-ICMw 是核心现象。
   - 意图漂移：**D_drift = 1 - f-ICMw**。
4. **公私分解**：
   - 公共组件：维度值可从模型训练分布中可预测恢复。
   - 私有组件：维度值对个体用户特有，模型可能提供通用默认值但无法恢复真实意图。
   - 分类是相对的：取决于任务类型、模型先验和说明上下文。
5. **不可逆意图损失定理（TIIL）**：
   - 假设单次交互、维度缺席编码、该维度是私有的。则没有解码器（即模型）能从固定载体-仅信息状态恢复该私有值，超出通用替换。
   - 信息论理由：数据不等式 → I(vₖ; g(P₋ₖ, M)) ≤ I(vₖ; P₋ₖ, M) ≈ 0。
   - 实际后果：模型扩展不能消除私有意图差距；编码失败是用户侧问题，模型改进无法补救。

## 6. 主要结论与发现

- **结构化意图编码提高对齐**：行为层表明协议结构化载体（5W3H）相比非结构化提示显著提高对齐分数，且增益源于编码完整性而非长度或措辞。
- **跨语言、跨模型泛化**：泛化层显示优势在三种语言和六个模型族中一致，弱模型更受益于结构化编码（弱模型补偿效应）。
- **协议层行为**：结构化意图具有类似协议的稳定特性，优于其他框架设计（CO-STAR, RISEN），并非模板技巧。
- **结构-保真度分裂是系统性的**：测量层发现，在中文中25.7%的输出落入分裂区（整体分=5但f-ICMw<0.8），英文中58.6%。人类评价者对这些输出给出的平均整体分仅3.12，而LLM评判给了5.0，证实分裂不是测量伪影。
- **公私不对称**：31.5%的单元格为公共机制，68.5%为私有机制。旅行规划领域公共率最高，商业/技术领域私有率最高。
- **权重容忍平台**：在权重中度偏差下保真度变化很小（容忍平台），但严重偏差下100%的单元格保真度下降（悬崖效应）。
- **不可逆意图损失定理**：私有意图一旦从载体缺失，模型扩展无法恢复，只能靠用户侧协议干预。
- **对AI系统架构的启示**：IST提出需要一个计算意图层，记录编码、恢复、漂移等，使交互可审计。
