# See and Switch: Vision-Based Branching for Interactive Robot-Skill Programming 

Petr Vanc, Jan Kristof Behrens, Václav Hlaváč, and Karla Stepanova 

Abstract — Programming robots by demonstration (PbD) is an intuitive concept, but scaling it to real-world variability remains a challenge for most current teaching frameworks. Conditional task graphs are very expressive and can be de-fined incrementally, which fits very well with the PbD idea. However, acting using conditional task graphs requires reliable perception-grounded online branch selection. In this paper, we present See & Switch , an interactive teaching-and-execution framework that represents tasks as user-extendable graphs of skill parts connected via decision states (DS), enabling conditional branching during replay. Unlike prior approaches that rely on manual branching or low-dimensional signals (e.g., proprioception), our vision-based Switcher uses eye-in-hand images (high-dimensional) to select among competing successor skill parts and to detect out-of-distribution contexts that require new demonstrations. We integrate kinesthetic teaching, joystick control, and hand gestures via an input-modality-abstraction layer and demonstrate that our proposed method is teaching modality-independent, enabling efficient in-situ recovery demonstrations. The system is validated in experiments on three challenging dexterous manipulation tasks. We evaluate our method under diverse conditions and fur-thermore conduct user studies with 8 participants. We show that the proposed method reliably performs branch selection and anomaly detection for novice users, achieving 90 .7% and 

87 .9% accuracy, respectively, across 576 real-robot rollouts. We provide all code and data required to reproduce our experiments at imitrob.ciirc.cvut.cz/publications/seeandswitch. 

I. Introduction 

Teaching robots new tasks from human guidance rather than code-based programming makes automation more ac-cessible to non-experts. In Learning from Demonstration 

(LfD), a robot derives its control policy by imitating the observed human behavior (e.g., kinesthetic teaching) instead of relying on hand-crafted code or trajectories [1], [2], [3]. Building on this idea, Programming by Demonstration 

(PbD) enables end-users to teach robots interactively through demonstrations, without writing any code or tuning parame-ters [4], [5], [1]. This demonstration-based approach is intuitive, but a key challenge is that a single demonstration’s validity is limited to the current state of the environment and its immediate neighborhood. In the face of significant changes, a fixed replay of the original demonstration may fail because of the demonstration’s implicit assumptions. Consider the following example. The robot is taught to measure a voltage with a probe by demonstrating how to approach the contact point. It might succeed when the contact point is accessible, but if it is hidden behind a closed door (see Fig. 1), the original behavior is inappropriate. In this paper, we focus on overcoming this limitation and enabling the robot to incrementally learn to handle the given task under different circumstances by leveraging iterative human feedback and grounding in the visual domain. Our solution builds on the concept of conditional skills :distinct task variants corresponding to alternative environ-mental states, such as various probe states. Recent PbD frameworks [6], [5], [1] represent such behaviors using task-graph models with multiple skill variants , where a skill variant defines a complete task solution as an end-to-end execution path from task start to completion. Each skill variant is composed of individual skill parts (trajectory segments). During execution, it is possible to switch at de-cision states (DS). At each DS, a suitable branch (successor skill part) is selected online based on sensory feedback [1], defining the executed variant. The Collaborative Incremental Programming (CIP) system by Eiband et al. [5] exemplifies this strategy: it incrementally constructs a task graph of individual skill parts from demonstrations, and an anomaly detector monitors the robot inputs to decide if execution is out-of-distribution (OOD) and if a new skill part is needed. However, a central challenge remains: selecting the appro-priate skill variant online given high-dimensional real-world observations. During incremental learning or unexpected fail-ures, the robot may encounter previously unseen situations (anomalies) that must be detected, and the user prompted

to provide a new demonstration. Existing systems typically rely on low-level internal signals, such as force sensing [5]. Vision-based approaches for skill branching are increasingly more popular [7], but many still require manually specified perceptual features. While proprioception reliably captures internal contact and motion states, it is inherently limited to what the robot can sense through its body, often only during interaction, whereas vision provides broader, contact-free access to environmental context across diverse scenarios. In this work, we extend task-graph-based PbD with a vision-grounded decision mechanism and human feedback to enable incremental, open-ended refinement. We introduce an image-based Switcher (Sec. III-D) that uses an eye-in-hand camera to evaluate the scene at decision states, to (i) select the appropriate skill variant, and (ii) detect novel contexts outside the current task graph. When such contexts occur, the system can (iii) incorporate situation-specific corrective demonstrations to expand the task graph via conditional branching. To support efficient refinement, we provide a modality-agnostic teaching interface that allows corrective demonstrations during execution via kinesthetic guidance, joystick control, or hand gestures. This enables users to add recovery behaviors online with minimal effort and without restructuring the existing task representation. Building on the CIP-style framework [5], our contributions are:  

> •

Automatic insertion of decision states (DS) triggered by (visual) anomaly detection [8] or user intervention, en-abling conditional branching for incremental refinement;  

> •

Online task-graph expansion via user-guided recovery demonstrations , adding new skill variants without mod-ifying the existing structure;  

> •

Modality-agnostic teaching (kinesthetic, joystick, ges-tures) for efficient in-situ recovery demonstrations. We evaluate the framework in a user study comparing teaching modalities in terms of feasibility and teaching efficiency. II. Related Work 

PbD has progressed from single-trajectory replay to in-teractive, graph-structured task models capturing conditional behavior. CIP learns task graphs with decision states and recovery branches from multiple demonstrations, using pro-prioceptive and force-based uncertainty for anomaly trig-gering and branch selection [8], [5], while Aayush et al. induce reactive behavior trees from one demonstration via a combination of learning and logic-based synthesis [9]. Neural Task Graph Networks infer latent graphs from a single video and execute unseen multi-step tasks [10]. We adopt the same high-level representation (user-editable task graph) but replace hand-crafted switching logic with a unified, image-based classifier that operates at decision-state context windows, enabling robust and automatic branch selection under ambiguous visual conditions. Execution monitoring and recovery have been addressed by learning recovery motions and visual monitors. Wu et al. encode anomaly recovery skills with Gaussian Process regression over multiple demonstrations [11]. Condition-NET and ILeSiA learn, respectively, symbolic precondi-tions/effects and visual risk scores from language and camera supervision to monitor execution [12], [13], while symbolic failure-detection pipelines learn human-interpretable predi-cates from multimodal signals [14]. Diffusion-PbD instead uses visual foundation models to extract keyframes and transfer waypoints to novel scenes, focusing on generalization rather than explicit, user-editable branching [15]. In contrast, our Switcher is trained per decision state on eye-in-hand images to jointly (i) classify among competing skill parts and (ii) flag OOD views within the shared embedding space, tightly coupling branch selection and anomaly detection. Vision foundation models are increasingly used as state encoders for manipulation and conditional control. DI-NOBot [16] leverages DINO [17] features for retrieval-based one-shot imitation and pixel-level alignment to novel objects. Similarly, we use DINO features within a decision-state-specific classifier to support efficient branch selection and anomaly detection, aligning visual representation learning with the discrete, incremental structure of task-graph exe-cution. 

III. Method 

We present a modality-agnostic system for programming 

conditional skills (multi-variant tasks) from user demon-strations. The approach integrates online execution mon-itoring and mechanisms for teaching recovery behaviors, formalized in the robot policy described in Sec. III-A). Our design retains the CIP [5] decision-state (DS) logic and anomaly-triggered insertion rule (Sec. III-B). We introduce (i) an input-modality-abstraction layer that unifies teaching interfaces (Sec. III-C), and ( ii ) a vision-based observation channel (eye-in-hand) capable of detecting anomalies and online selection of the most suitable skill continuation at each DS: the Switcher, Sec. III-D. The core idea of our method is to enable incremental user-guided task definition and to simplify task-part selection and anomaly detection by conditioning decisions on perceptions at the DS and only deciding among relevant task-part alternatives. This effectively avoids the need for global scene understanding.

E. Proposed DINO-based Switcher for DS estimation 

To implement DS-local state estimation in the Switcher, we use self-supervised DINO vision transformers [17] as frozen feature extractors. Given an eye-in-hand image z,the backbone outputs a set of patch embeddings {xk}Kk=1 ,

xk ∈ Rd. We convert these patch embeddings into aclass prediction over competing skill parts using one of the following strategies: 

a) Prototype inference (mean).: We form an image representation either by mean pooling ( mean ) or by concate-nating patch embeddings ( concat ). For each class ci, we compute a prototype by averaging the training representations of that class. At inference time, we assign the query image to the class with maximum cosine similarity to its prototype. 

b) Multi-instance learning (MIL).: We treat patch em-beddings as a set of instances and learn an attention-based MIL head that aggregates {xk} into a class prediction. 

c) Attention-gated features (attn).: To emphasize infor-mative regions, we use the last-layer [CLS] -to-patch self-attention weights as patch importance scores. We apply either 

hard gating (discard low-attention patches) or soft gating (down-weight them), and reduce multiple attention heads by mean or max before aggregation. 

d) Anomaly detection.: We use the same DINO embed-ding space for novelty detection by computing an anomaly score from the query image’s similarity to the DS train-ing set (e.g., based on the maximum cosine similarity to known classes) and flagging an anomaly when the score crosses a threshold (set via a training-score percentile 

percentile_keep ∈ (0 , 1] ). 

IV. Experimental Setup 

We evaluate the proposed system on a dataset collected during our user study where users were asked to teach a robot manipulation tasks on the Robothon Electronic Task Board [19], Fig. 1, bottom left. Our experimental setup includes a Franka Robotics Panda robot with a RealSense D455 camera mounted in an eye-in-hand configuration, as shown in Fig. 1 (top left). 

A. Robothon Manipulation Skills 

We consider three manipulation tasks on the taskboard: (1) Peg pick, (2) Probe measure, and (3) Cable wrap. The Scenarios are detailed in Fig. 5. For each task, we vary the initial state in a way that requires an adapted manipulation. This can be, for example, a different initial pose for the peg ( lA, lB in Fig. 5) or a closed door over the probe measurement point. In our experiments, users teach/program these tasks to (i) evaluate the system usability, and at the same time (ii) collect the evaluation dataset D.

B. User study dataset configuration 

We collect the dataset D = (Ddemo , R) during the user study to evaluate the proposed Switcher (Sec. III-D) for two objectives: (i) target skill-part estimation (branch selection) and (ii) anomaly detection. Ddemo is the set of demonstrations and R the set of roll-outs. Each participant was first introduced to the system terminology and task setup (Fig. 5). To build intuition for the demonstration workflow for our single manipulator, we asked participants to perform the selected tasks single-handedly before actually teaching the robot. The users were then asked to perform the task using the given teaching method once. The task is reset to another variation, rendering the original demonstration unsuccessful. The user would then flag an anomaly as soon as the culprit is visible in the camera image. From that state on, the user demonstrates the remaining task parts (up to 3 variants) till the task end. The dataset covers the 3 manipulation tasks defined in Sec. IV-A. The study involved 8 participants (age 24–40). (∼22M) and large (∼300M) backbones.

C. Novice user study: teaching efficiency and task success 

We report replay success rates (manual switching) and average demonstration lengths (teaching effort) in Tab. II. Replay success is highest for Peg pick (up to 99 .2% 

with joystick), while Cable wrap is hardest overall ( 48 .8–

58 .1% across modalities). Since switching is manual in this experiment, most failed task success cases were slip grasps or missing end actions, rather than switching errors. We analyze failed decision success cases separately in Sec. V-B. A recurring failure mode is limited observability, e.g., DS triggered when the camera does not capture the door state (open/closed), making correct branch selection impossible from images alone. Demonstration length serves as a proxy for teaching effort: as expected, kinesthetic teaching is consistently fastest (19.5– 24.2s) and stays efficient even for Cable wrap (23.2s), while joystick and gestures become substantially longer on orientation-sensitive tasks (up to 61.4 s and 73.5 s for Cable wrap ). The high joystick performance on Peg pick (short and accurate) is consistent with the task requiring mostly top-down grasps with minimal wrist reorientation, whereas Probe measure demands precise end-effector alignment. Ges-ture teleoperation was particularly sensitive to hand sensor reliability. Example trajectories are shown in Fig. 7. This is visible in Fig. 7, where the gesture trajectory becomes notably irregular during the measurement phase while re-orienting the gripper. Experienced users tend to produce fewer unnecessary actions and errors [25], whereas novices often provide suboptimal trajectories [26]. When partici-pants repeated demonstrations, their performance typically improved, leading to shorter and more reliable executions [27]. Finally, the trajectory shapes differ systematically across modalities: kinesthetic demonstrations tend to be smooth and direct, joystick trajectories are more segmented due to incremental command inputs, and gesture trajectories can be noisier during fine orientation adjustments (Fig. 7). 

VI. Conclusion and Discussion 

We presented a conditional PbD framework that represents tasks as branching skill parts and supports incremental exten-sion via decision states (DS). A vision-based Switcher selects the appropriate branch at each DS or flags observations as outside the demonstrated contexts, i.e., an anomaly. 

a) Switcher performance and practical implications: 

On the user-study DS windows, performance varies due to noisy and partially observable data: for branch selection, only about half of DS instances achieve high test accuracy (e.g., 31/78 DS models exceed 90% for dinov2 small attn ). By filtering out cases where, for example, the camera does not capture the correct features, the overall accuracy increases from 81.7% to 90.7%. In contrast, anomaly de-tection is more stable (43/78 DS models exceed 90%). In a controlled label-growth experiment, dinov2 small attn remains robust as the number of competing classes increases (93.6% at 5 classes, and 90.8% at 6 classes), supporting use in larger DS branch sets. 

b) User study findings: Non-expert participants suc-cessfully taught conditional tasks where kinesthetic teach-ing produced the shortest demonstrations (19.5–24.2s) and remained efficient even for Cable wrap (23.2s), while joy-stick and gesture teleoperation became much longer for the harder tasks (up to 61.4s and 73.5s, respectively). Replay success was highest for Peg pick (up to 99.2% with joystick), while Cable wrap was the most challenging overall (48.8– 58.1%), with kinesthetic teaching achieving the best success rate. Taken together, the results support the practical use of kinesthetic teaching when available, while also showing that gesture-based teaching remains a viable alternative for modality-independent task programming. 

c) Limitations and open problems: The main limitation is observability: switching fails when the discriminative scene information is not visible to the eye-in-hand camera, motivating multi-view sensing or viewpoint control. Per-formance can also degrade under strong visual distractors. Finally, enabling re-joining of branches after the decisive variation could reduce graph growth and better support long tasks with local corrections.
