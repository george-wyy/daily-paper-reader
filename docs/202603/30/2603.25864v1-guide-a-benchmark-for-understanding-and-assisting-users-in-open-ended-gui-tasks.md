## 摘要
图形用户界面（GUI）智能体具有辅助用户与复杂软件（如 PowerPoint、Photoshop）交互的潜力。虽然之前的研究主要集中在通过点击和按键实现用户操作的自动化，但这种范式忽略了人类意图，即用户在保持自主性的同时，重视探索、迭代和完善其想法的能力。为了超越自动化并迈向协作，GUI 智能体必须理解用户正在做什么以及为什么这样做。我们推出了 GUIDE（GUI 用户意图检测评估），这是一个评估 AI 模型在开放式 GUI 任务中感知用户行为、推断意图并提供辅助能力的基准测试。GUIDE 包含来自 10 种软件、120 名新手用户演示的 67.5 小时屏幕录像，并配有出声思维（think-aloud）叙述。GUIDE 定义了三个任务：(i) 行为状态检测，(ii) 意图预测，以及 (iii) 帮助预测，旨在测试模型识别行为状态、推理目标以及决定何时及如何提供帮助的能力。对八个最先进的多模态模型的评估显示，所有模型都表现不佳，在行为状态和帮助预测方面的准确率仅为 44.6% 和 55.0%。然而，提供用户上下文显著提高了性能，将帮助预测的准确率提升了高达 50.2 个百分点，突显了结构化用户理解在有效辅助中的关键作用。我们的数据集可在 https://guide-bench.github.io 获取。

## Abstract
Graphical User Interface (GUI) agents have the potential to assist users in interacting with complex software (e.g., PowerPoint, Photoshop). While prior research has primarily focused on automating user actions through clicks and keystrokes, this paradigm overlooks human intention, where users value the ability to explore, iterate, and refine their ideas while maintaining agency. To move beyond automation and toward collaboration, GUI agents must understand what users are doing and why. We introduce GUIDE (GUI User Intent Detection Evaluation), a benchmark that evaluates AI models on their ability to perceive user behavior, infer intent, and provide assistance in open-ended GUI tasks. GUIDE consists of 67.5 hours of screen recordings from 120 novice user demonstrations with think-aloud narrations, across 10 software. GUIDE defines three tasks - (i) Behavior State Detection, (ii) Intent Prediction, and (iii) Help Prediction that test a model's ability to recognize behavior state, reason about goals, and decide when and how to help. Evaluations across eight state-of-the-art multimodal models reveal that all models struggled, achieving only 44.6% and 55.0% accuracy on behavior state and help prediction. However, providing user context significantly improved the performance, raising help prediction by up to 50.2pp, highlighting the critical role of structured user understanding in effective assistance. Our dataset is available at https://guide-bench.github.io.

### GUIDE 论文深度总结：开放式 GUI 任务中的用户理解与辅助基准

#### 1. 核心问题与整体含义（研究动机和背景）
当前的图形用户界面（GUI）智能体研究主要集中在**任务自动化**（如通过点击和按键完成特定指令），但这忽略了真实世界中用户与复杂软件（如 Photoshop、Excel）交互的本质。在开放式任务中，用户往往需要经历探索、试错和方案迭代，他们更看重**自主权（Agency）**而非单纯的效率。
**核心问题**：现有的 AI 模型缺乏对用户行为状态、潜在意图以及“何时/如何提供帮助”的深度理解。论文旨在将 GUI 智能体从“被动执行者”转变为“主动协作助手”，使其能够感知用户的认知上下文。

#### 2. 论文提出的方法论
论文提出了 **GUIDE (GUI User Intent Detection Evaluation)** 基准，其核心思想是将用户理解建模为一个三阶段的推理框架：
*   **行为状态检测 (Behavior State Detection)**：识别用户当前处于哪种认知/操作阶段。论文构建了一个包含 9 种状态的分类体系（如：规划、执行、挫折、调试、评估等），这些状态被归类为四个高层阶段：计划、执行、问题解决和评估。
*   **意图预测 (Intent Prediction)**：在给定操作序列下，推理用户的短期目标（例如“我想对齐这些对象”）。
*   **帮助预测 (Help Prediction)**：
    1.  **需求检测**：判断用户当前是否陷入困境并需要帮助。
    2.  **内容预测**：确定最合适的辅助形式（如解释功能、建议替代方案或修复错误）。
*   **数据标注技术**：采用“人机协作”模式。利用用户录制时的“出声思维（Think-aloud）”叙述作为真值来源，先由 Gemini-2.5-Pro 生成初始标注，再由人工进行多轮验证和精细化修正。

#### 3. 实验设计
*   **数据集 (GUIDE Dataset)**：
    *   **规模**：67.5 小时屏幕录像，来自 120 名新手用户的演示。
    *   **覆盖范围**：涵盖 10 种主流软件（Photoshop, Figma, PowerPoint, Premiere Pro, Excel 等），涉及 40 个开放式任务。
    *   **多样性**：每个任务由 3 名不同用户完成，捕捉多样化的策略和错误模式。
*   **Benchmark 任务设置**：
    *   **离线设置**：模型观察完整的视频片段进行推理。
    *   **在线设置**：模拟实时助手，模型按 25%、50%、75%、100% 的进度逐步接收视频流。
*   **对比方法**：评估了 8 个最先进的多模态大模型 (MLLMs)，包括闭源模型（Gemini-2.5-Pro/Flash, GPT-4o/mini, Claude-4.5-Sonnet）和开源模型（Qwen3-VL, InternVideo2.5, InternVL3）。

#### 4. 资源与算力
*   **算力说明**：论文主要侧重于对现有 SOTA 模型的 **Zero-shot（零样本）** 评估，因此未详细描述大规模模型训练的算力消耗。
*   **推理资源**：实验使用了各模型的公开 API 或官方 Checkpoints。对于开源模型（如 InternVideo2.5-8B），通常需要单张或多张 A100/H100 级别的 GPU 进行推理。

#### 5. 实验数量与充分性
*   **实验规模**：在 1.8K 个行为检测样本、1.3K 个意图预测样本和 1K 个帮助预测样本上进行了详尽测试。
*   **消融/对比实验**：设计了多种上下文输入配置（如“仅视频” vs “视频 + 行为状态” vs “视频 + 行为 + 意图”），以验证结构化上下文对辅助决策的贡献。
*   **充分性评价**：实验设计非常充分且具有客观性。通过引入 Multi-Binary Accuracy (MBAcc) 等严格指标，排除了模型通过随机猜测或偏见获胜的可能性。

#### 6. 论文的主要结论与发现
*   **模型表现普遍欠佳**：在没有额外上下文的情况下，所有模型在行为状态检测（最高 44.6%）和帮助预测（最高 55.0%）上都表现吃力。
*   **难以识别“负面”信号**：模型经常将用户的“挫折”或“调试”行为误判为正常的“执行”或“探索”，说明模型对重复点击、犹豫等细微的求助信号不敏感。
*   **上下文是关键**：提供结构化的用户上下文（如告知模型用户当前的意图）能显著提升性能。例如，加入意图信息后，某模型的帮助预测准确率提升了 50.2 个百分点。
*   **时间上下文的重要性**：在在线流式评估中，随着视频信息的增加，模型的预测准确率稳步上升，证明了长时记忆和时序推理对主动助手的必要性。

#### 7. 优点（亮点）
*   **视角独特**：不同于以往关注“专家演示”和“任务自动化”的基准，GUIDE 专注于“新手用户”和“协作辅助”，更贴近真实的人机交互场景。
*   **高质量数据**：结合了屏幕录制、键盘鼠标事件和**出声思维叙述**，为理解用户心理状态提供了极佳的 Ground Truth。
*   **认知科学支撑**：行为分类法参考了 Norman 的“行动七阶段”和 Bloom 的认知层次结构，具有坚实的理论基础。

#### 8. 不足与局限
*   **视觉信号的局限性**：某些心理状态（如纯粹的思考）在视觉上可能表现为静止，仅靠视频帧可能无法完全区分“思考”与“发呆”。
*   **模型偏见**：实验发现模型倾向于假设用户是“高效且正确的”，这种先验偏见导致其在检测用户错误和求助需求时召回率较低。
*   **应用限制**：目前仅评估了模型的“判断”能力，尚未在真实的实时交互系统中测试这些判断如何转化为具体的 UI 干预动作。
