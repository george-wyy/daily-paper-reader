## 摘要
多模态大语言模型（MLLMs）显著推动了移动代理的发展，但主动式移动辅助仍面临挑战，因为代理必须在确定如何协助之前决定何时干预。现有系统通常将这两个决策统一集成到基于MLLM的流程中，导致保守干预过滤与全面辅助生成之间的目标错位，以及在代理应保持静默时的冗余推理。为解决这些限制，我们提出预推理感知框架（PRPF），这是一种建立在感知先于推理理念上的两阶段框架。PRPF引入轻量级的多模态主动感知器（MPP）用于干预门控和上下文压缩，并且仅在需要干预时激活主动代理推理器（PAR）。在ProactiveMobile基准上的实验表明，与ProactiveMobile基线相比，PRPF显著降低了误触发率（FTR），同时提高了成功率（SR）和推理效率。

## Abstract
Multimodal large language models (MLLMs) have substantially advanced mobile agents, yet proactive mobile assistance remains challenging because agents must decide \emph{when} to intervene before determining \emph{how} to assist. Existing systems often implement these two decisions within a unified MLLM-based pipeline, leading to goal misalignment between conservative intervention filtering and comprehensive assistance generation, as well as redundant inference when the agent should remain silent. To address these limitations, we propose the \textbf{Pre-Reasoning Perception Framework (PRPF)}, a two-stage framework built on perceiving before reasoning. PRPF introduces a lightweight Multimodal Proactive Perceptor (MPP) for intervention gating and context compression, and activates the Proactive Agent Reasoner (PAR) only when intervention is warranted. Experiments on the ProactiveMobile benchmark show that PRPF substantially reduces false trigger rates (FTR) while improving success rates (SR) and inference efficiency over the ProactiveMobile baseline.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：当前主动移动代理（Proactive Mobile Agents）需要同时解决“何时干预”（when to intervene）和“如何协助”（how to assist）两个子任务。现有系统通常将两者统一集成在单一的 MLLM（多模态大语言模型）流程中，导致两个关键问题：
  - **目标错位**：干预决策需要保守、高鉴别力以降低误触发，而辅助生成需要广泛的推理和灵活的内容生成，两者在同一模型内难以兼顾。
  - **推理低效**：即使代理应保持静默（不干预），完整推理流程仍被执行，造成不必要的计算开销，尤其在长上下文中。
- **整体含义**：该论文主张将“感知”与“推理”解耦，提出先进行轻量级干预判断，仅在需要时启动大规模推理，从而在降低误触发的同时提升成功率与推理效率，推动主动移动代理迈向实用化。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：提出 **预推理感知框架（Pre-Reasoning Perception Framework, PRPF）**，基于“感知先于推理”原则，架构层面解耦“何时干预”与“如何协助”。
- **两阶段架构**：
  - **阶段一：轻量级多模态主动感知器（MPP）** —— 快速判断是否应触发推荐，并压缩上下文（预测 Top-K 候选功能场景）。
    - **快慢双通道设计**：慢通道（Slow Channel）捕捉长程行为模式（用户偏好），通过注意力池化全局视觉状态并与文本上下文结合；快通道（Fast Channel）捕捉短程动态（最近4帧截图）。
    - **跨通道交互**：使用交叉注意力（Cross-Attention）增强快慢通道信息融合，得到融合表示 \( \mathbf{z} \)。
    - **两个任务头部**：
      - 触发门控头（Trigger-Gating Head）：输出干预概率 \( p_{\text{trig}} \)，若小于阈值 \(\tau\) 则过滤掉样本。
      - 功能场景预测头（Function-Scenario Prediction Head）：输出各场景概率，选取 Top-K 候选功能构建压缩的功能集 \(\tilde{\mathcal{F}}\)。
  - **阶段二：主动代理推理器（PAR）** —— 仅在 MPP 判断为需要干预时激活，使用大模型（Qwen3.5-9B）进行深度推理，生成结构化的推荐建议，包括 UI 摘要、思维链（thinking）和最终功能调用序列。
- **训练目标**：
  - **MPP 训练**：联合优化触发门控损失（Focal Loss，处理类别不平衡）和场景预测损失（BCE），总损失 \( \mathcal{L}_{\text{MPP}} = \mathcal{L}_{\text{trig}} + \lambda \mathcal{L}_{\text{scenario}} \)。
  - **PAR 训练**：先通过监督微调（SFT）学习结构化输出格式，再使用 GRPO（Group Relative Policy Optimization）强化学习进一步稳定决策和提升生成质量，奖励函数包括准确率、格式合规性和校准项。
- **推理流程**：测试时，每个样本先经过 MPP，若 \( p_{\text{trig}} < \tau \) 则直接输出空推荐（静默），否则将压缩后的功能集与完整上下文传递至 PAR 生成最终输出。

- PRPF 在 ProactiveMobile 基准上取得 **最优整体性能**：常规平均 SR 达到 41.15%，FTR 降至 7.21%，显著优于最强单阶段基线 Qwen3.5-9B+Proactive（SR 34.56%, FTR 13.49%）。

- **瓶颈揭示**：多模态理解仍是主要限制，即使 PRPF 也仅达到 17.19% SR，强调需要更强 GUI 理解能力。
